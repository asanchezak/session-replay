================================================================================
                          KEY DATA FLOWS
================================================================================

  RECORDING:
    User action → Content Script (capture.ts)
                → RECORD_EVENT message → Service Worker (orchestrator.ts)
                → POST /v1/workflows/record → Backend → PostgreSQL

  EXECUTION (agent loop):
    Service Worker → CAPTURE_PAGE_CONTEXT → Content Script → page state
    Service Worker → POST /v1/agent/{id}/poll → AgentService.poll()
                    → (AI consult if enabled) → returns AgentDecision
    Service Worker → EXECUTE_AGENT_COMMAND → Content Script (replay.ts)
    Service Worker → POST /v1/agent/{id}/result → AgentService.report_result()

  DASHBOARD:
    Frontend hooks → GET /v1/runs, /v1/workflows, etc. → Backend API → DB
    5s poll for waiting_for_user runs → InterventionModal on match

  OBSERVABILITY:
    All layers → Seq (http://localhost:8082) via structured logging
    Extension → POST /v1/debug/log → Debug endpoint

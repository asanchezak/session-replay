## Recommendations

1. **Fix `real-world-flow` cross-page recording** (P2): The content script does not re-attach after cross-origin navigation in headless Chrome. A service worker `webNavigation.onCommitted` listener that injects content script on navigation would fix this permanently rather than relying on the retry.

2. **Stabilize SW `GET_STATE` after restart** (P2): The `sw-lifecycle` test logs `GET_STATE response: no-response`. The SW should queue and drain pending message requests on wake-up rather than silently dropping them.

3. **Add run-with-params execution path** (P3): The `run-with-params` endpoint creates runs that can only be executed when the extension SW polls and picks them up. Consider adding a test that simulates the full path: create via params → SW executes → verify substitution in navigation events.

4. **Add timing guards to auth-flow** (P2): The `onInstalled` storage-seeding test is flaky because chrome.storage.session writes aren't immediately visible. Add a `waitFor` polling the storage key before proceeding.
